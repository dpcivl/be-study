# Java 백엔드 개발자 전환 커리큘럼

> **최종 목표:** 배포되어 실제로 운영되는 LLM 백엔드 서비스 1개 완성
> **대상:** SW 경력 2년 5개월 + 임베디드 HW 1년 (프로그래밍 경험 있음 / **자바는 처음**)
> **기간:** 평일 1.5~2h + 주말 몰아서 병행 기준 약 4~5개월
> **하루 세션 비율:** 코딩 실습 7 : 개념 학습 2 : 기록(TIL/커밋) 1

---

## 진행 방식

- **파일 하나 만들어 직접 타이핑 → 실행 → 눈으로 확인 → 왜 그런지 짚기**의 반복. 개념 설명을 길게 읽기보다, 코드를 쳐보며 익힌다.
- 매일 GitHub에 커밋하고 배운 것을 TIL로 남긴다. 꾸준함 자체가 면접에서 강력한 신호가 된다.
- 각 과의 **완료 기준**을 통과해야 다음 과로 넘어간다. 시간이 아니라 결과물로 진도를 관리한다.
- 프로그래밍 개념(변수·반복·조건)은 이미 아니 빠르게 가고, **자바 특유의 것(타입, 객체지향)** 에 시간을 더 쓴다.

# PART 1. 자바 기초 (4~5주) ← 자바가 처음이므로 여기부터 탄탄히

## 1과. 자바 시작 — 설치 · 첫 실행 · 변수와 타입
- **목표:** 자바 코드를 써서 화면에 출력하고, 변수에 값을 담는다.
- **내용:** JDK 설치, `main`부터 실행되는 구조, class라는 상자, `System.out.println`, 변수 타입(String/int/double), 문자열 이어붙이기.
- **실습:** `Hello.java`, `Variables.java` — 본인 값으로 바꿔 출력.
- **완료 기준:** 자바 파일을 만들어 실행하고, 변수 3종을 선언·출력할 수 있다.

## 2과. 조건문 · 반복문
- **목표:** 흐름 제어를 자바 문법으로 익힌다.
- **내용:** if/else, switch, for, while, 비교·논리 연산자, 증감.
- **실습:** 구구단 출력, 1~100 합·짝수만 출력 등 작은 프로그램.
- **완료 기준:** for와 if를 조합해 조건에 맞는 값만 골라 출력할 수 있다.

## 3과. 메서드 & 배열
- **목표:** 반복되는 로직을 함수로 묶고, 여러 값을 다룬다.
- **내용:** 메서드 정의(반환 타입·파라미터·return), 메서드 호출, 배열 선언·순회.
- **실습:** 배열의 합·평균·최댓값을 구하는 메서드 작성.
- **완료 기준:** 파라미터를 받아 값을 반환하는 메서드를 직접 만들고 호출한다.

## 4과. 클래스와 객체 (OOP 1) ← 자바의 핵심
- **목표:** 자바가 세상을 객체로 보는 방식을 익힌다.
- **내용:** class와 object의 차이, 필드, 생성자, 메서드, `new`로 객체 만들기, 캡슐화(private + getter/setter).
- **실습:** `Book`, `Account` 같은 클래스를 만들어 객체를 여러 개 생성·사용.
- **완료 기준:** 필드·생성자·메서드를 가진 클래스를 만들어 객체로 다룬다.

## 5과. 상속 · 인터페이스 · 다형성 (OOP 2) ← Spring 이해의 토대
- **목표:** Spring이 서 있는 객체지향 개념을 잡는다.
- **내용:** 상속(extends), 오버라이딩, 인터페이스(implements), 다형성, 추상화가 왜 유용한지.
- **실습:** 인터페이스 하나에 여러 구현체를 만들어 갈아끼우는 예제.
- **완료 기준:** "인터페이스로 프로그래밍한다"가 무슨 뜻인지 예시로 설명할 수 있다. (DI 이해의 밑밥)

## 6과. 컬렉션 & 제네릭
- **목표:** 실무에서 가장 많이 쓰는 데이터 구조를 익힌다.
- **내용:** List/Set/Map 특성과 선택 기준, 제네릭(`List<String>`)의 의미.
- **실습:** Map으로 단어 개수 세기, List로 데이터 모아 순회.
- **완료 기준:** List와 Map을 상황에 맞게 골라 쓸 수 있다.

## 7과. 람다 · 스트림 · Optional
- **목표:** 모던 자바의 데이터 처리 방식을 익힌다.
- **내용:** 람다, 메서드 참조, Stream API(map/filter/collect/groupingBy), Optional로 null 안전하게 다루기.
- **실습:** 리스트를 스트림으로 필터·변환·집계.
- **완료 기준:** 반복문 로직을 스트림으로 바꿀 수 있고, Optional을 `.get()` 없이 처리한다.

## 8과. 예외 처리 & 동시성 기초
- **목표:** 백엔드에서 반드시 겪는 에러·동시성 개념을 잡는다.
- **내용:** try-catch, Checked vs Unchecked 예외, 커스텀 예외, 스레드/스레드풀(ExecutorService) 개념.
- **실습:** 예외를 일부러 발생시켜 잡아보고, 스레드풀로 여러 작업 동시 실행.
- **완료 기준:** "스레드풀이 왜 필요한가"를 말로 설명할 수 있다. (WAS·비동기 이해의 밑밥)

## 9과. 모던 자바 문법 정리
- **목표:** 요즘 코드가 간결해지는 문법을 정리한다.
- **내용:** record, var, switch 표현식, 텍스트 블록.
- **실습:** 앞서 만든 데이터 클래스를 record로 재작성.
- **완료 기준:** DTO/값 객체를 record로 만들 수 있다.

## 10과. Gradle 빌드 도구
- **목표:** 프로젝트 빌드·의존성 관리 흐름을 이해한다.
- **내용:** build.gradle 구조, 의존성 추가, 빌드 라이프사이클, 실행 가능한 jar.
- **실습:** 프로젝트를 Gradle로 구성해 빌드·실행.
- **완료 기준:** 새 라이브러리를 의존성에 추가해 쓸 수 있다.

# PART 2. Spring Boot 코어 (3~4주) ← 면접 최다 출제 구간

## 11과. Spring 핵심 개념 (DI / IoC)
- **목표:** 스프링이 무엇을 대신 해주는지 이해한다.
- **내용:** IoC 컨테이너, 빈(Bean), 의존성 주입(생성자 주입 권장 이유), @Component/@Service/@Repository. (5과 인터페이스 개념이 여기서 빛을 발함)
- **실습:** Spring Boot 프로젝트 생성(start.spring.io), 빈을 만들어 주입받아 실행.
- **완료 기준:** "왜 new 대신 DI를 쓰는가"를 테스트 용이성 관점에서 설명한다.

## 12과. Spring MVC & REST API 설계
- **목표:** HTTP 요청을 받아 응답하는 기본기를 만든다.
- **내용:** @RestController, 매핑 애노테이션, @RequestBody/@PathVariable/@RequestParam, ResponseEntity, HTTP 상태코드, REST 설계 관례.
- **실습:** 메모리에 저장하는 간단한 CRUD API(Todo API) 제작.
- **완료 기준:** Postman으로 CRUD 4종을 다 호출해 정상 동작을 확인한다.

## 13과. 레이어드 아키텍처 · 검증 · 전역 예외 처리
- **목표:** 실무 표준 구조로 코드를 정리한다.
- **내용:** Controller-Service-Repository 분리, DTO와 Entity 분리, Bean Validation(@Valid), @RestControllerAdvice 전역 예외 처리, 통일된 에러 응답.
- **실습:** 12과 API를 3계층으로 리팩터링 + 검증·에러 응답 추가.
- **완료 기준:** 잘못된 요청에 상황별 상태코드와 통일된 에러 JSON을 반환한다.

# PART 3. 데이터 계층 (2~3주)

## 14과. 관계형 DB & SQL 기초
- **목표:** 데이터를 실제로 저장·조회하는 토대를 만든다.
- **내용:** MySQL/PostgreSQL 설치, 테이블 설계, PK/FK, 인덱스 개념, 기본 SQL(SELECT/JOIN/WHERE/GROUP BY).
- **실습:** 스키마를 직접 설계하고 SQL로 CRUD 실행.
- **완료 기준:** JOIN이 들어간 조회 쿼리를 작성할 수 있다.

## 15과. JPA / Hibernate & Spring Data JPA
- **목표:** 자바 객체와 DB 테이블을 매핑해 다룬다.
- **내용:** @Entity, 연관관계 매핑, 지연/즉시 로딩, JpaRepository, 쿼리 메서드.
- **실습:** 13과 API의 저장소를 메모리에서 실제 DB(JPA)로 교체.
- **완료 기준:** 엔티티 2개 이상을 연관관계로 매핑해 저장·조회한다.

## 16과. 트랜잭션 & 실무 단골 이슈
- **목표:** 면접 단골인 데이터 계층 함정을 직접 겪는다.
- **내용:** @Transactional 원리, 영속성 컨텍스트, **N+1 문제**와 해결(fetch join), DTO 조회.
- **실습:** 일부러 N+1을 발생시켜 로그로 확인 후 개선.
- **완료 기준:** N+1이 무엇이고 어떻게 잡는지 예시로 설명한다.

## 마무리 관점

면접관은 상업적으로 성공한 제품이 아니라, **왜 이렇게 설계했고 · 부하와 에러를 어떻게 다뤘고 · 다시 하면 뭘 바꿀지**를 유창하게 말할 수 있는 사람을 원한다. 배포·운영해본 사이드 프로젝트 1개 + 그 의사결정을 설명하는 능력이면 "상용화 경험 없음"은 충분히 뒤집을 수 있다.

특히 AI 회사 지원 시 **"LLM도 알고, 그걸 서빙하는 백엔드도 직접 짜서 운영해봤다"** 는 조합이 강력하므로, PART 5 프로젝트를 반드시 LLM과 엮는다.
